# Contributing to PocketBook

Thank you for your interest in contributing to PocketBook! We welcome contributions from the community.

## How to Contribute

### Reporting Bugs

1. Check existing [issues](https://github.com/sanojvenu/PocketBook-v1.0/issues) to avoid duplicates.
2. Open a new issue with a clear title and description.
3. Include steps to reproduce, expected vs actual behavior, and screenshots if applicable.

### Suggesting Features

1. Open a [feature request](https://github.com/sanojvenu/PocketBook-v1.0/issues/new) issue.
2. Describe the feature, its use case, and any relevant context.

### Submitting Code

1. **Fork** the repository.
2. **Create a branch** from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes** following the code style of the project.
4. **Test your changes** locally (web and Android if applicable).
5. **Commit** with a clear message:
   ```bash
   git commit -m "feat: add your feature description"
   ```
6. **Push** and open a **Pull Request** against `main`.

## Development Setup

See the [README](README.md) for detailed setup instructions.

## Code Style

- Use TypeScript with strict types.
- Follow existing patterns for component structure.
- Use Tailwind CSS utility classes for styling.
- Keep components focused and reusable.

## Commit Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` — New features
- `fix:` — Bug fixes
- `docs:` — Documentation changes
- `style:` — Code style (formatting, no logic change)
- `refactor:` — Refactoring
- `test:` — Adding or updating tests
- `chore:` — Maintenance tasks

## Security

If you discover a security vulnerability, please see [SECURITY.md](SECURITY.md) for responsible disclosure guidelines. **Do not** open a public issue for security vulnerabilities.
